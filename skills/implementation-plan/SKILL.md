---
name: implementation-plan
description: Turn a request into a clean, low-risk execution plan
---

1. Restate the task clearly in engineering terms
2. Define expected outcome and success criteria
3. Break into small, ordered steps
4. Identify dependencies, assumptions, and edge cases
5. Highlight risks and where things may break
6. Define validation steps (commands, outputs, UI checks)
7. Keep plan minimal, practical, and aligned with existing patterns
